# vworld 3D 분석 15종 — 입력 파라미터 명세

사용자가 분석을 요청하면 AI가 **이 표대로 필요한 파라미터를 먼저 물어본 뒤** `vworld map 3dsim --analysis <type>`로 HTML을 생성한다.

파라미터 성격:
- 📍 **위치(필수, 전 종 공통)**: 주소 또는 좌표 `lon,lat` → CLI `--address "..."` / `--center lon,lat`로 주입
- ⚙️ **옵션값**: 생성 HTML의 입력 필드 기본값. 물어서 안내(현재는 화면에서 조정, 향후 CLI 확장 가능)
- 🖱️ **지도 인터랙션**: 생성된 지도에서 마우스로 직접(영역 그리기·지점 클릭·경로). AI가 대신 못 함 → 사용자에게 절차 안내

## AI 동작 규칙 (요청 처리 흐름)

1. 사용자가 "○○ 분석해줘"라고 하면 → 위 표에서 종을 식별.
2. **AskUserQuestion**으로 묻는다:
   - 📍 위치(주소 또는 좌표) — 항상 첫 질문(필수).
   - ⚙️ 해당 종의 핵심 옵션값(선택지가 명확한 것 위주). 답 없으면 샘플 기본값 사용.
3. `vworld map 3dsim --analysis <type> --address "..."`(또는 `--center`)로 HTML 생성.
4. 🖱️ 지도 인터랙션이 필요한 종은, 생성 후 "지도에서 …를 그리거나 클릭하세요" 절차를 안내.
5. (옵션) "결과만" 요청 시 → 헤드리스 브라우저로 분석 실행 후 결과 데이터만 추출(slope 등 지점형은 자동, 영역형은 좌표 필요).

## 현재 CLI 주입 범위 (정직)
- **지금 CLI에 반영되는 것**: 📍 위치(`--address`/`--center`)뿐.
- ⚙️ 옵션값은 생성 HTML의 입력 필드 **기본값**으로 들어가며, 사용자가 화면에서 조정. (향후 `--opt key=value` 식 CLI 확장 가능)
- 🖱️ 인터랙션은 브라우저에서만 가능.
