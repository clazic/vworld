## 1. 현황 분석 (전부 실측 검증)

`vworld ned`는 115 오퍼레이션을 레지스트리에 등록(`ned --list` → 115). `NedOp`는 `apinum/kind/endpoint_op/cat1/cat2/name`만 — **요청변수 메타 없음**. `--param k=v` 임의 패스스루는 **이미 존재**(`NedArgs.params`).

### 1.1 kind 분포·호출 가능 여부 (실측)
| kind | 개수 | 현재 CLI | 실측 결과 |
|------|------|---------|----------|
| **data** | 46 | `ned <op> --pnu … --param k=v` | ✅ 작동(JSON). geomFilter/attrFilter/columns/numOfRows/pageNo는 **전용 플래그 없음 → `--param`으로만** |
| **wfs** | 33 | `ned <op> --pnu/--bbox/--all/--by-hjd --param` | ✅ 작동(GeoJSON). 동일하게 세부 필터는 `--param` |
| **wms** | 36 | `run_ned`이 **거부**(`data_cmds.rs:514`) | ⚠️ **미해결** — 아래 §1.3 |

### 1.2 거부 배경 (코드+설계 확인)
설계(`.omc/plans/2026-06-16-…구현계획.md` §1.1.2): *"WFS·data는 데이터형, **WMS는 이미지형(Step 4)으로 라우팅**"* + *"옵션 미수집 정직 표기·1차 필수키+`--param` 패스스루"*. 즉 WMS는 의도적으로 미뤄둔 미완성 — 거부 메시지가 가리키는 "이미지형 경로"가 `/ned/wms/*`를 실제 구현하지 않음.

### 1.3 ⚠️ NED WMS 스펙 **미확정** (C1·C2 실측 — 핵심 리스크)
- **`/ned/wms/{op}`(naive 파라미터)는 빈 이미지 반환**: `getBuildingAgeWMS/getLandUseWMS/getIndvdLandPriceWMS/CtnlgsSpceService` 모두, crs(4326·900913)·bbox·영역 무관하게 **동일한 4401바이트 PNG, 비흰색 픽셀 0%(완전 공백)**. `service/request/version` 부착 여부도 무관(동일 응답).
- **대조: `/req/wms?layers=lt_c_uq111&crs=EPSG:4326&bbox=…`는 정상**(124KB, 비흰색 99%). → **키는 WMS 가능하나, `/ned/wms/{op}` 경로가 naive 파라미터로 데이터를 안 줌.**
- **결론**: NED 주제 WMS의 **실제 작동 호출법이 확정되지 않았다.** 가설: (a) NED 주제도는 `/req/wms?layers=<NED레이어명>`로 접근, (b) `/ned/wms/{op}`에 미발견 필수 파라미터(styles/layers/data/특정 domain) 필요, (c) 키의 NED WMS 권한/도메인 문제. **이전 rev1의 "PNG 512×512 확인"은 빈 이미지였음(거짓 통과).**

### 1.4 요청변수 표 파싱 주의 (스크랩 견고성)
상세페이지 단순 `<td>` 파싱은 **에러코드 표를 오집**(실측 확인). "요청변수" 표를 **헤더/섹션 기준으로 정확히 지정**해야 함 → 스크랩은 탐색(구조 확인) 후 구현으로 2분.

## 2. RALPLAN-DR 요약

### Principles
1. **실측 우선** — 미확정 스펙(NED WMS)은 작동 호출을 **실측으로 박제한 뒤** 구현. 가정 금지.
2. **자기완결·단일 진입점** — `ned` 명령 확장(신규 명령 최소화).
3. **빌드 결정성** — `build.rs`는 **커밋된 파일만** 읽고 **네트워크 절대 미사용**. 스크랩은 개발자 1회 수동.
4. **기능 우선 + 발견성** — 모든 요청변수 전달 가능(`--param`) + 메타로 도움말·필수검증.
5. **단일 출처 무결성** — 요청변수 메타는 기존 `ned_catalog.tsv` 단일 출처에 통합(이원화 금지).

### Decision Drivers (Top 3)
1. **NED WMS 36개 작동 호출법 확정**(미확정 리스크 해소) — 최우선 게이트.
2. **115개 요청변수 메타**(검증·도움말).
3. **빌드 결정성·단일출처·회귀 안전**.

### Viable Options
**Option A — `ned` 확장: (WMS 디스커버리→구현) + 요청변수 메타(TSV 통합) + 검증/도움말 (권장)**
- 장점: 단일 진입점으로 115개 커버, 요청 정확 부합, 빌드 결정성 유지.
- 단점: WMS 디스커버리 결과에 따라 WMS 구현 범위 가변(리스크를 게이트로 관리).

**Option B — data/wfs 메타만 + WMS는 보류(거부 메시지만 개선)**
- 장점: 확정된 79개만 안전하게. 최소 리스크.
- 단점: WMS 36개 미해결 → "모두 구현" 미달. 단, **WMS가 공개 경로로 불가하면 B가 정직한 종착**.

**Option C — op별 타입 플래그 풀 구현** — clap 폭발·과설계. → 무효.

→ **채택: Option A, 단 WMS는 디스커버리 게이트 통과 시에만 구현.** 게이트 실패(공개 작동 경로 없음) 시 해당 부분만 **B로 폴백**(정직 표기). C는 무효.

## 3. 구현 계획

### 단계 −1 — NED WMS 라이브 스펙 디스커버리 (게이트, 담당: Opus 분석 + Sonnet) **[C1·C2]**
WMS 구현 착수 **전 필수**. 다음을 실측해 **작동하는 전체 쿼리스트링을 계획서/주석에 박제**:
1. `/req/wms?layers=<NED레이어명>` 경로 가설 검증 — NED 주제도의 실제 layer명을 상세페이지/문서에서 확보해 `/req/wms`로 GetMap 시도, 내용(비흰색%) 확인.
2. `/ned/wms/{op}` 필수 파라미터 전수 탐색 — 상세페이지 "요청변수" 표(정확 파싱)에서 styles/layers/data/format/domain 등 필수값 확인 후 재시도.
3. 도메인 변이(무도메인 키/등록도메인) 영향 확인.
- **게이트 판정**: (성공) 비흰색 픽셀 > 5%인 **작동 호출 확정** → 단계2 진행, 그 쿼리스트링을 박제. (실패) 공개 작동 경로 없음 → WMS는 **B 폴백**(거부 메시지에 정확한 사유·대안 명시, 36개는 "미지원(공개 경로 부재)"로 정직 표기).

### 단계 0 — 요청변수 스크랩 (탐색→구현, 담당: Sonnet) **[스크랩 견고성]**
- (0a) **탐색**: 상세페이지 1~3건 HTML을 실제로 열어 "요청변수" 표의 DOM 구조(섹션 헤더·표 클래스·열 순서) 확정. **에러코드 표와 구분**.
- (0b) `scripts/scrape_ned_params.py`(표준 라이브러리, 크로스플랫폼): 115 `apiNum` 순회 → "요청변수" 표만 파싱 → op별 `[{name, required, type, default, desc}]`.
  - 레이트리밋 대비: 요청 간 지연(0.3s), 실패 재시도 1회, 실패 op 목록 출력.
  - 산출: `skills/data/ned_params.tsv`(또는 기존 TSV 확장 — 단계1 참조). **이 파일은 git 커밋**.
- **수작업 폴백 정직 표기**: 파싱 불가 op는 빈 메타로 두지 말고 `parse_failed=true`로 표기(Principle 1 정직성).

### 단계 1 — 메타를 단일 출처(TSV)에 통합 + 코드젠 (담당: Sonnet) **[C3·M1]**
- **단일 출처 확정**: 요청변수 메타를 **기존 `skills/data/ned_catalog.tsv`에 7번째 컬럼(params: JSON 문자열)**으로 통합(별도 파일·이원화 금지). 빈 메타/parse_failed는 컬럼에 명시 마커. (1:1 매핑 자동 보장 — 같은 행이라 드리프트 불가.) build.rs 드리프트 가드를 7컬럼으로 갱신.
- `build.rs`: **커밋된 TSV만 읽음(네트워크 절대 미사용)**, `cargo:rerun-if-changed` 유지. `NedOp.params: &'static [NedParam]` 코드젠.
- `ned_registry.rs`: `NedParam{name,required,r#type,default,desc}` 추가·직렬화.
- **회귀**: 기존 36/33/46 드리프트 가드 + 115 카운트 단언 유지. `ned --list` JSON에 `params` 추가는 **append-only**(기존 필드 불변).

### 단계 2 — NED WMS 구현 (단계−1 게이트 통과 시, 담당: Sonnet) **[C1·C2]**
- 단계−1에서 박제한 **확정 쿼리**대로 구현. WMS 경로/파라미터를 `NedBuilder`(또는 적정 빌더)에 분기 추가.
- `NedArgs` 신규 공통 플래그: `--width`(기본 512), `--height`(512), `--img-format`(image/png), `--transparent`, `--crs`(기본=단계−1 실측값). 나머지는 `--param`.
- 출력: 이미지 바이트 저장(`output::save_bytes`), **기본 출력명 `ned_<op>.png`**(연속 호출 덮어쓰기 방지).
- **내용 검증(디코딩 불요 — 신규 크레이트 회피)**: 빈 NED WMS는 파라미터 무관하게 **고정 4401바이트·동일 콘텐츠 해시**를 반환(실측). 단계−1에서 이 "공백 핑거프린트(바이트길이+SHA256)"를 박제 → 응답이 핑거프린트와 일치하면 **실패 처리**(빈 이미지 거부). + 본문 XML 에러 검사. **이미지 디코딩 라이브러리(`image`/`png`) 미사용**이 기본. (선택: 더 엄밀한 비흰색 픽셀 검증이 필요하면 `image` 크레이트 도입 — **CLAUDE.md상 사용자 승인 필요**, 기본 계획엔 미포함.)

### 단계 3 — 요청변수 검증·도움말 (담당: Sonnet) **[M2·M3]**
- `ned <op> --params`: 해당 op 요청변수 표 출력. **신규 bool 플래그 `--params`(`NedArgs`에 추가) — 기존 `--param`(Vec, 패스스루)과 별개**(혼동 방지 위해 헬프에 명확 구분, 필요 시 별칭 `--show-params` 검토).
- **필수값 판정**: 메타 required 충족 여부를 **전용 플래그(`--pnu/--bbox/--width/…`) AND `--param`에 들어온 키 둘 다 스캔**해 판정(`--param pnu=…`도 충족 인정). 메타에 없는 키는 패스스루 허용.
- **플래그 적용 매트릭스**(혼란 방지): 신규 공통 플래그가 kind별로 적용/무시되는지 명시 — `--width/--height/--img-format/--transparent`=WMS 전용(타 kind에서 무시+경고), `--crs`=WMS/WFS단건(— `--by-hjd`는 내부 4326 고정이라 `--crs` 지정 시 **경고 후 무시**).

### 단계 4 — 빌드·검증 (담당: Haiku/Sonnet)
- `cargo build --release` + `cargo test`.
- 스폿체크: (WMS 게이트 통과 시) WMS 3종 → **내용 있는 이미지**, data 2·wfs 1 → JSON/GeoJSON, `--params` 출력, 필수 누락 에러.

## 4. 사전 점검 (Pre-mortem)
1. **NED WMS 공개 작동 경로가 끝내 없음** → 단계−1 게이트가 차단, WMS는 B 폴백(정직 표기). 36개를 억지 구현해 빈 이미지 양산하는 사태 방지.
2. **빈 이미지 거짓 통과**(C2) → **디코딩 불요 공백 핑거프린트**(단계−1 박제한 4401B 길이+SHA256)와 일치하면 실패 처리. 이미지 크레이트 미사용으로 의존성 갭·승인 게이트 회피.
3. **build.rs 네트워크 의존으로 CI 파괴**(C3) → 불변식: build.rs는 커밋 TSV만. 스크랩은 수동 1회·산출물 커밋. CI에 네트워크 불필요.
4. **TSV/메타 정합성 깨짐**(M1) → 단일 출처(TSV 컬럼) 또는 1:1 가드로 빌드 실패 유도.
5. **필수값 오판정으로 `--param` 사용자에 거짓 에러**(M2) → 전용 플래그+`--param` 동시 스캔.

## 6. ADR
- **Decision**: `ned` 확장 — (1) **NED WMS 작동 호출을 디스커버리 게이트로 확정 후** 이미지 구현(미확정 가정 금지), (2) 요청변수 메타를 **단일 출처(TSV)**에 통합·build.rs 코드젠(네트워크 미사용), (3) `--params` 도움말 + 전용플래그·`--param` 동시 인정 필수검증. `--param` 패스스루 안전망 유지.
- **Drivers**: WMS 작동 확정, 요청변수 발견성·검증, 빌드 결정성·단일출처.
- **Alternatives**: B(WMS 보류) — WMS 공개 경로 부재 시의 폴백으로 채택. C(타입 플래그) — 과설계 무효.
- **Why chosen**: 실측으로 WMS 빈 이미지(거짓 통과)·빌드 네트워크 리스크를 확인 → 게이트·내용검증·결정성으로 차단하며 "요청변수 보고 구현"을 충족.
