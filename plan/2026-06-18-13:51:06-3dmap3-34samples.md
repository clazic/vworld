## 1. 배경 / 현황 분석 (근거 확인 완료)

`vworld`는 Rust 단일 바이너리 CLI다. 3D 분석·시뮬레이션 템플릿을 `include_str!`로 **컴파일 타임 임베드**한다.

- 레지스트리: `src/cli/embed_cmds.rs:97` `const ANALYSES: &[(&str,&str,&str,&str)]`
  - 튜플 = `(키, 한글설명, API버전, include_str!(HTML))`
  - 현재 15종: **1.0 분석** 11종 + **2.0 가시화** 4종
- HTML 원본: `src/cli/tool3d_samples/*.html` — vworld 공식 샘플, `@{apikey}` 플레이스홀더 포함
- 사용: `vworld map --analysis <키>` → `run_analysis()`(`embed_cmds.rs:219`)
  - `@{apikey}` → 실제 키(+`&domain=`) 치환
  - `</head>` → `TOSS_STYLE` 주입 (`embed_cmds.rs:163,253`)
  - `--center`/`--address` 지정 시 `SAMPLE_CENTER` 리터럴(`embed_cmds.rs:185`) 치환으로 재중심
  - `--analysis list` → 목록 출력

### 1.1 신규 34종의 출처 / 추출 경로 (실측 검증 완료)

3.0 가이드 페이지의 `#listDiv`는 AJAX로 채워진다. 추출 경로를 curl로 실증했다:

1. **목록**: `POST /dev/v4dv_opnmapexam_s001.do` `body: ppsCtt=API012&searchTab=tab2`
   → `<li>` 34개, 각 `goDetail('EXAID_…')` + `class="con" title="설명"` 포함 (34건 정확히 확인)
2. **개별 소스**: `POST /dev/v4dv_opnmapexam_s002.do` `body: ppsCtt=API012&exaIde=<EXAID>`
   → `<textarea id="sampleCtt">`에 **HTML 엔티티 인코딩된 완전한 HTML 소스**
   → 디코드(`&lt;`→`<`, `&#034;`→`"`, `&amp;`→`&`) 시 그대로 실행 가능한 자기완결 HTML

### 1.2 3.0 샘플의 구조적 특성 (샘플 1건 실측)

| 항목 | 값 | 함의 |
|------|-----|------|
| 스크립트 | `//map.vworld.kr/js/webglMapInit.js.do?version=3.0&apiKey=@{apikey}` | 버전 **3.0**, `@{apikey}` 동일 → 기존 주입 로직 그대로 사용 |
| 지도 컨테이너 | `<div id="vmap">` | TOSS `#vmap` 셀렉터와 **정확히 일치** → 스타일 주입 OK |
| `</head>` | 존재 | TOSS 주입 OK |
| API 클래스 | `vw.MapController`, `vw.ol3.*` | 1.0/2.0(`vw.Map`+tool3d)과 **다른 API 체계** |
| 초기 좌표 | `vw.ol3.CameraPosition`(`homePosition`/`initPosition`), `CoordZ` 리터럴 **전무** | `SAMPLE_CENTER` 재중심이 **34종 전건 미적용(0%)** — `--center`/`--address`가 조용히 무시됨 (M1) |
| 지도 div 인라인 style | `<div id="vmap" style="width:100%;height:350px;left:0px;top:0px">` | TOSS `#vmap{height:70vh!important}`가 인라인 height를 이김(CSS specificity). 단 `left/top/position`류 인라인은 TOSS 대응 규칙 없음 → **34종 전수 스캔 필요** (M2) |
| 외부 리소스 URL | `//map.vworld.kr` (프로토콜 상대경로) | ⚠️ **메모리 규칙 위반** — `(src|href)="//"` 전수 `https://` 정규화 필요 |

## 2. RALPLAN-DR 요약

### Principles
1. **자기완결 유지** — 모든 샘플은 컴파일 타임 임베드(`include_str!`), 런타임 네트워크 의존 없음.
2. **기존 패턴 일관성** — 신규 34종도 `ANALYSES` 튜플 + `run_analysis()` 경로를 그대로 재사용. 새 명령/구조 신설 금지.
3. **생성 HTML 규칙 준수** — 외부 리소스 절대경로(`https://`) 강제, TOSS 디자인 주입(메모리 규칙).
4. **키 전역 유일성** — `--analysis <키>`는 버전 무관 단일 키 조회. 기존 15키와 충돌 없는 신규 34키.
5. **재현 가능한 추출** — 34개 HTML을 스크립트로 일괄 다운로드·디코드·정규화(수작업 금지).

### Decision Drivers (Top 3)
1. 34개 HTML 소스의 **정확한 추출·디코드·정규화 자동화** (수작업 시 오류·누락 위험).
2. 기존 1.0/2.0과의 **통합 방식** — 동일 배열 vs 분리 구조.
3. 3.0 고유 특성(`//` 프로토콜 상대경로, **재중심 전건 미적용**, 인라인 style) **수용 범위와 사용자 가시 동작 정의**.

### 결정: 3.0 재중심 처리 (M1 해결)
- **본 작업 범위**: `run_analysis`에서 분석 키의 API 버전이 `"3.0"`일 때 `--center`/`--address`가 지정되면 **명시적으로 무시**하고, ① `eprintln!` 경고 출력, ② 출력 JSON 메타에 `"center_applied": false`(+사유) 표기. → 더 이상 거짓 성공(`"ok":true`인데 좌표 미반영) 없음.
- **차기(Follow-up)**: 3.0 `vw.ol3.CameraPosition`에 좌표 주입(`ol2d_init_js`의 `ol.proj.fromLonLat([lon,lat])` 패턴 재사용)으로 실제 재중심 구현.
- 1.0/2.0 기존 15종은 `SAMPLE_CENTER` 치환 경로 **무변경**(회귀 없음).

### Viable Options

**Option A — 동일 `ANALYSES` 배열에 `"3.0"` 버전으로 49종 통합 (권장)**
- 장점: 기존 `run_analysis`/`list_analyses`/`--analysis list` 무수정 재사용. 사용자 경험 일관. 최소 변경.
- 단점: 배열 49줄로 증가. `list` 출력이 길어짐(버전 그룹 헤더로 완화).

**Option B — 3.0 전용 별도 배열 + 별도 서브커맨드(`map --analysis3` 등)**
- 장점: 3.0 API 체계 분리 명시. 재중심 미지원 등 3.0 특수성 격리.
- 단점: 명령 표면 이원화, 코드 중복(list/lookup 로직 복제), 사용자 혼란. **드라이버 2와 충돌.**

**Option C — 런타임 다운로드(임베드 안 함)**
- 장점: 바이너리 크기 유지.
- 단점: **자기완결 원칙(Principle 1) 정면 위배.** 오프라인 불가, vworld 페이지 변경에 취약. → 무효.

→ **채택: Option A.** B는 일관성/단순성 드라이버에 반하고, C는 핵심 설계 원칙 위배로 무효.

## 7. ADR

- **Decision**: 34개 3.0 공식 샘플 HTML을 스크립트로 일괄 추출·정규화해 `tool3d_samples/v3_*.html`로 임베드하고, 기존 `ANALYSES` 배열에 `"3.0"` 버전으로 통합 등록한다.
- **Drivers**: 자동화된 정확 추출, 기존 패턴 일관성, 자기완결 원칙.
- **Alternatives considered**: (B) 별도 배열/서브커맨드 — 일관성·단순성 저해로 기각. (C) 런타임 다운로드 — 자기완결 원칙 위배로 무효.
- **Why chosen**: 최소 변경으로 기존 `run_analysis`/`list`/TOSS/키주입 경로를 그대로 재사용, 사용자 경험 일관.
- **Consequences**: 바이너리 크기 증가(텍스트 HTML 34종, 단계 0에서 합계 정량화 — 샘플 1건 ~1.2KB 기준 수십~수백KB 추정), `list` 출력에 버전 그룹 헤더 추가(단계 2d 실제 구현). 3.0은 `--center`/`--address` 재중심 **본 범위 미구현 → 무시+메타 표기로 명시**(거짓 성공 제거), 실제 주입은 Follow-up. 3.0 API 체계(`vw.MapController`/`vw.ol3`)가 1.0과 달라 단일 배열에 누적되는 분기 부채는 인지하되 현 규모에선 통합이 단순성 우위.
- **Follow-ups**: ① 3.0 `vw.ol3.CameraPosition` 좌표 주입으로 실제 재중심 구현, ② README/사용법 49종 갱신, ③ 34종 인라인 style `position` 동반 샘플 발견 시 TOSS 보정 규칙 추가.
