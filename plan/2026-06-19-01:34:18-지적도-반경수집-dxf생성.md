## 1. 현황·실현성 (전부 실측, Critic 반영)

### 1.1 구성요소 상태
| 요소 | 현재 | 비고 |
|------|------|------|
| 주소→좌표 | `geocode_point(g, addr)` → **EPSG:4326 하드코딩**(data_cmds.rs:283) | **기본 4326이면 시그니처 변경 불요**(C1 회피) |
| 연속지적도 WFS | `--bbox` 경로 존재, **bbox 5토큰 `minx,miny,maxx,maxy,EPSG:xxxx`**(ned_params.tsv:81), 4326은 (lat,lon) 축 | totalFeatures 제공(cap 응답에도 1567 노출 — 실측) |
| 영역 전수수집 | `--all`은 **PNU 접두 분할 전용**(bbox 아님) | **bbox 격자 엔진은 완전 신규**(M1) |
| DXF 출력 | 없음 | **신규**. MTEXT 요구 → R12 불가, **R2000(AC1015) 타깃** |

### 1.2 핵심 제약 (실측)
1. **WFS 1000 cap**: 자란만로 476 1km(2km×2km)=1,567필지, 1회 1,000만 반환. 2×2 격자 시 각 415/114/334/808(<1000) 확인. 경계중복 → **PNU dedup 필수**.
2. **좌표계**: **사용자 지정 기본 EPSG:4326**. geocode_point가 4326 반환·WFS srsName 기본 4326 → **TM(5186/5187) 변환·WFS 5187 출력 불요**(Critic C2·C3 회피). `--crs 5186/5187` 오버라이드는 **Follow-up**(geocode_point crs 인자화 + WFS 출력 실측 스파이크 + 5187 변환기 필요 — 기본 경로엔 불포함).
3. **DXF 버전**: 사용자 요구 **MTEXT 라벨**은 R13+ 전용 → **DXF R2000(AC1015)** 타깃(MTEXT·LWPOLYLINE·`$DWGCODEPAGE` 지원). R12/LWPOLYLINE 혼용 표기는 폐기.
4. **한글 인코딩**: 사용자 요구 **CP949** → DXF `$DWGCODEPAGE=ANSI_949` + 텍스트 바이트 CP949. **기존 의존성 `encoding_rs`(EUC-KR/UHC) 재사용**(새 크레이트 0).

### 1.3 사용자 지정 DXF 기본값 (QGIS DXF 내보내기 미러링)
| 설정 | 기본값 | 구현 매핑 |
|------|--------|----------|
| 좌표계 | **EPSG:4326** | geocode·WFS·DXF 좌표 모두 4326(경위도) |
| 인코딩 | **CP949** | `$DWGCODEPAGE=ANSI_949`, 한글 바이트 CP949(encoding_rs) |
| 심볼 모드 | **심볼 레이어 심볼 유형**(Symbol Layer Symbology) | 필지 윤곽=레이어 `PARCEL` LWPOLYLINE(심볼 레이어 단위 지오메트리 출력) |
| 심볼 스케일 | **1:1000** | 텍스트 높이·선폭을 1:1000 도면 기준으로 산정(스케일 팩터 노출) |
| 라벨 | **MTEXT** | 지번을 MTEXT 엔티티(레이어 `JIBUN`), 필지 centroid |
| 범위 필터 | **현재 범위와 교차 객체 내보내기** | bbox와 **교차(intersect)**하는 필지 포함(WFS bbox 기본 의미와 일치) |

## 2. RALPLAN-DR 요약

### Principles
1. **실측 우선** — 미검증 가정 금지. 기본 4326로 좌표계 CRITICAL 회피, 오버라이드는 스파이크 후 Follow-up.
2. **자기완결** — DXF writer 외부 의존성 0(encoding_rs만 기존). DWG는 범위 외(ODA 변환 안내).
3. **신규 명확화** — bbox 격자 엔진·DXF writer는 신규임을 명시(재사용 과장 금지).
4. **데이터 무결성** — 격자 cap 초과 시 분할(BFS 단일 워커풀)·PNU dedup, 누락 0.
5. **사용자 기본값 준수** — 4326/CP949/MTEXT/심볼레이어/1:1000/교차내보내기.

### Decision Drivers (Top 3)
1. cap 누락 없는 bbox 전수수집(BFS 격자 + dedup).
2. 사용자 DXF 기본값(4326·CP949·MTEXT·R2000) 정확 구현.
3. 기본 경로의 좌표계 단순성(4326)으로 리스크 최소화.

### Viable Options
**Option A — `ned` 확장: `--address`/`--radius`/`--dxf`(+옵션) WFS 격자수집→DXF (권장)**
- 장점: 단일 명령, geocode·WFS bbox 재사용(격자/DXF만 신규), 다른 WFS 주제도 확장.
- 단점: ned 플래그 증가(이미 20+). `--dxf` WFS 게이팅 필요.

**Option B — 전용 서브커맨드 `cadastre`**
- 장점: 발견성, ned 플래그 혼재 회피(ned는 WMS/WFS/data/all/by-hjd 모드 혼재로 게이팅 버그 잦음).
- 단점: WFS 인프라 일부 중복.

→ **채택: Option A**, 단 `--dxf`/`--address`/`--radius`는 **WFS 계열에서만 동작(타 kind 지정 시 에러)**로 게이팅(`--all`/`--by-hjd`의 `kind != Wfs` 에러 패턴 따름). B의 발견성 장점은 인정하나, geocode+WFS+DXF 재사용과 단일 명령 UX가 우선.

## 3. 구현 계획

### 단계 −1 — 선행 검증 스파이크 (게이트, 담당: Opus/Sonnet) **[C3·M1]**
착수 전 1회 실측해 계획에 박제:
- **신규 `wfs_count_bbox(bbox_5토큰)` 헬퍼**(M3): 기존 `wfs_count`(data_cmds.rs:1210)는 **pnu 전용**(`.set("pnu", …)`)이라 bbox 카운트에 못 씀. **bbox + maxFeatures=1**로 보내 `totalFeatures`만 받는 헬퍼 신규. 스파이크로 bbox 쿼리도 totalFeatures를 주는지 확인(실측: 1567 노출 확인).
- **`properties.pnu` 노출 확인**(dedup 키): `getCtnlgsSpceWFS` feature가 `properties.pnu`를 실제로 갖는지 1건 확인(없으면 dedup 키를 `id`로). 기존 dedup(data_cmds.rs:1310)이 pnu 전제.
- **`output=application/json` 강제**: bbox 쿼리도 JSON(features 배열) 반환하도록 — leaf 파싱이 `features` 가정(data_cmds.rs:1295).
- ⚠️ **`ned_wms_bbox`(data_cmds.rs:524) 재사용 금지** — 양 분기 동일한 **no-op**라 축순서 안 바꿈. bbox 축순서는 §단계0의 명시 분기로 직접 처리.
- (오버라이드용, Follow-up 게이트) `srsName=EPSG:5187` 출력이 실제 TM 미터인지 — **기본 4326 경로엔 불요**, 오버라이드 도입 시에만.

### 단계 0 — bbox 격자 harvest 엔진 (신규, 담당: Sonnet) **[M1·M2·M3]**
- `NedArgs`에 `--address`, `--radius`(기본 1000m), `--dxf <파일>`, `--crs`(기본 **EPSG:4326**), `--encoding`(기본 **cp949**), `--symbol-scale`(기본 **1000**) 추가. (모두 **WFS 전용** 게이팅.)
- 중심: `--address` → `geocode_point(g, addr)`(4326). 반경→**bbox(4326 degree)**: Δlat=radius/111320, Δlon=radius/(111320·cos(lat)).
- **bbox 문자열 조립**(M2): WFS는 `minx,miny,maxx,maxy,<CRS>` 5토큰. **4326은 축순서 (ymin,xmin,ymax,xmax)** → `"<latmin>,<lonmin>,<latmax>,<lonmax>,EPSG:4326"`. CRS별 축순서 분기 + 단위테스트.
- **BFS 격자 분할 — 레벨 동기(level-synchronous)**(M3 정정): `run_jobs`(worker.rs:13)는 **배치형**(고정 `Vec<J>` 받고 채널 닫음) — in-flight에 동적 추가 불가. 따라서 **프론티어 레벨마다 `run_jobs` 1회**(count 프로브) → `totalFeatures ≥ 1000`인 칸을 4분할해 **다음 프론티어** 구성 → 반복 → 마지막에 leaf 칸들의 수집을 `run_jobs`. **기존 `harvest_wfs_all`(data_cmds.rs:1236~)의 레벨별 2-페이즈 구조를 그대로 따름**. 동시성 상한은 레벨별 `run_jobs`로 유지(전역 단일 풀 불가능 — 정정). 분할 깊이 상한(pnu 경로의 16과 정렬 검토, 기본 8) 도달 시에도 ≥1000이면 **경고 + 부분수집 정직 표기**.
- **교차 필터·dedup**: WFS bbox는 교차 필지 반환(사용자 "교차 내보내기" 기본과 일치). 격자 경계 중복 → **PNU(feature properties/id) dedup**.

### 단계 1 — DXF writer R2000(AC1015) (신규·자기완결, 담당: Sonnet) **[M4 + 사용자요구]**
- `src/dxf.rs`: **GeoJSON FeatureCollection → DXF R2000(AC1015)**. 외부 크레이트 0(`encoding_rs`만 기존).
  - 구조: HEADER(`$ACADVER=AC1015`, `$DWGCODEPAGE=ANSI_949`, `$INSUNITS` 등) + TABLES(LAYER: `PARCEL`,`JIBUN`) + ENTITIES + 핸들(R2000 필수, 단조 증가).
  - 필지 폴리곤 각 ring → **LWPOLYLINE**(레이어 `PARCEL`, closed flag 70=1). **MultiPolygon 다중 파트·내환(구멍)**: 각 ring을 독립 닫힌 LWPOLYLINE으로 출력(윤곽선만, 면 채움/HATCH 미지원 — 한계 표기). 내환도 동일 레이어 독립 LWPOLYLINE.
  - 지번/PNU → **MTEXT**(레이어 `JIBUN`, 필지 centroid). **텍스트 높이(4326 degree 명시 변환, M2)**: `height_deg = (paper_mm/1000 × scale_denom) / 111320` (위도축 기준, 예 2.5mm·1:1000 → 2.5m/111320 ≈ 2.25e-5°). 경도축 이방성(×cos φ 압축)은 **시각용으로 허용**(측량 정합은 5186/5187 오버라이드 Follow-up). 단위테스트로 대상 위도에서 비퇴화 높이 단언.
    - ⚠️ centroid는 단순 평균이라 오목/구멍 필지에선 라벨이 폴리곤 밖에 떨어질 수 있음(v1 허용, 한계 표기).
  - **인코딩 CP949**: MTEXT/문자열 바이트를 encoding_rs EUC-KR(UHC=CP949)로 인코딩, 파일 전체 CP949로 기록.
  - **심볼 레이어 심볼 유형**: 필지 윤곽을 단일 심볼레이어(선)로 출력 — 레이어 `PARCEL`에 LWPOLYLINE(현 단계는 단색·기본선; 지목별 색/스타일은 Follow-up).
- `--dxf` 지정 시 수집 결과를 DXF 저장, 아니면 기존 JSON/GeoJSON.

## 6. ADR
- **Decision**: `ned`(WFS 전용 게이팅)에 `--address`/`--radius`/`--dxf`/`--crs`/`--encoding`/`--symbol-scale` 추가 — geocode(4326) 중심 → **BFS bbox 격자**로 1000 cap 우회·PNU dedup 전수수집 → 자기완결 **DXF R2000(AC1015)** writer(필지 LWPOLYLINE + 지번 **MTEXT**, **CP949**, 심볼레이어/1:1000, 교차 내보내기). 기본 좌표계 **EPSG:4326**.
- **Drivers**: cap 누락 없는 수집, 사용자 DXF 기본값 정확, 기본 4326의 좌표계 단순성(CRITICAL 회피).
- **Alternatives**: B(전용 서브커맨드) 발견성 우위이나 재사용·단일UX로 기각. R12 DXF는 MTEXT 미지원이라 R2000으로 상향.
- **Why chosen**: 4326 기본이 Critic C1/C2/C3(geocode crs·5187 변환·WFS 5187 출력)를 회피하고 사용자 요구와도 일치. 신규는 격자엔진·DXF로 한정.
- **Consequences**: 4326 degree 좌표라 CAD 측량 정합은 사용자 재투영 책임(5186/5187 오버라이드는 Follow-up·스파이크 필요). DXF는 윤곽선만(면 채움/지목 스타일 범위 외). 도심 폭증은 깊이상한·부분수집 정직 표기.
- **Follow-ups**: `--crs 5186/5187` 오버라이드(geocode_point crs 인자화+3호출자 수정+5187 출력 스파이크+`lonlat_to_tm5187`), 지목별 색/스타일, 원형 `--circle` 클립, 면 HATCH, DWG 자동변환(ODA).

## 7. 사용법 (Usage) — 완성 후 기준

### 기본: 주소 반경 연속지적도 → DXF
```bash
# 자란만로 476 인근 1km 연속지적도를 DXF로 (기본 4326 / CP949 / MTEXT / 1:1000 / 교차)
vworld ned getCtnlgsSpceWFS --address "자란만로 476" --radius 1000 --dxf jaranman.dxf
```
- 동작: 주소 지오코딩(4326) → 중심±1km bbox → **BFS 격자 분할로 1000건 cap 우회** 전수수집 → PNU 중복제거 → DXF(R2000/AC1015) 저장.
- 출력 메타(JSON): 중심좌표, 격자 수·최대깊이, 수집·dedup 필지 수, DXF 경로·좌표계·인코딩.

### 옵션
| 옵션 | 기본 | 설명 |
|------|------|------|
| `--address <주소>` | — | 중심 주소(지오코딩). 없으면 `--bbox` 직접 지정 |
| `--radius <m>` | 1000 | 중심에서 정사각 bbox 반폭(미터) |
| `--dxf <파일>` | — | DXF 저장(미지정 시 GeoJSON/JSON 출력) |
| `--crs <EPSG>` | EPSG:4326 | 좌표계. 측량용 미터는 `EPSG:5186`(중부)/`5187`(동부) *(Follow-up)* |
| `--encoding <enc>` | cp949 | DXF 텍스트 인코딩(한글 지번) |
| `--symbol-scale <n>` | 1000 | 심볼/문자 크기 기준(1:n 도면) |

### 다른 활용 예
```bash
# bbox 직접 지정(주소 없이)
vworld ned getCtnlgsSpceWFS --bbox "37.49,127.02,37.52,127.06" --dxf parcels.dxf

# 좌표 데이터만(GeoJSON) — DXF 없이
vworld --raw ned getCtnlgsSpceWFS --address "자란만로 476" --radius 500 > parcels.geojson

# 측량용 미터좌표(Follow-up, 5187 동부원점)
vworld ned getCtnlgsSpceWFS --address "자란만로 476" --radius 1000 --crs EPSG:5187 --dxf survey.dxf
```

### DWG가 필요하면
DXF 생성 후 무료 **ODA File Converter**로 DXF→DWG 변환(CLI는 DXF까지 지원, DWG는 독점 포맷이라 외부 변환).

> ※ 본 섹션은 **구현 완료 후** 동작 기준. 미구현 상태에서는 동작하지 않음.
